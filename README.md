# E-Commerce SQL Analysis and Power BI

## Project Overview
This project analyzes an e-commerce dataset using SQL to extract insights about customer behavior, product performance, payment patterns, customer satisfaction, loyalty, and time-based sales trends.

The analysis covers end-to-end exploratory data analysis (EDA) and intermediate SQL techniques such as CTEs, window functions, ranking, and segmentation.

---

## Dataset Description
The dataset includes order-level e-commerce transactions with the following features:

- Customer information (Customer_ID, Age, Gender, City)
- Order details (Order_ID, Date, Quantity, Total_Amount)
- Product information (Product_Category, Unit_Price, Discount_Amount)
- Customer behavior (Device_Type, Session_Duration, Pages_Viewed)
- Satisfaction metrics (Customer_Rating, Delivery_Time_Days)
- Loyalty indicator (Is_Returning_Customer)

---

## Analysis Structure

### 1. Data Exploration & Data Quality (EDA)
Initial exploration of the dataset including:
- Row count validation
- Null value detection
- Duplicate record analysis
- Data type and structure checks

---

### 2. Customer Demographics Analysis
Analysis of customer distribution based on:
- Age distribution
- Gender breakdown
- City-based customer activity

---

### 3. Product & Category Analysis
Evaluation of product categories using:
- Total revenue
- Order count
- Units sold
- Average order value
- Customer rating

Includes segmentation using ranking and performance classification.

---

### 4. Payment & Device Behavior Analysis
Analysis of customer purchasing behavior based on:
- Payment methods
- Device types
- Payment-device combinations
- Revenue contribution of each segment

---

### 5. Customer Satisfaction & Loyalty Analysis
Evaluation of:
- Average customer ratings
- Delivery time impact
- Returning vs new customer behavior
- Customer segmentation based on value and behavior

---

### 6. Time Series & Revenue Analysis
Analysis of business performance over time:
- Monthly revenue trends
- Order volume changes
- Customer growth
- Average order value trends
- Cumulative revenue (running total)
- Seasonality patterns

---

## Key SQL Techniques Used
- GROUP BY aggregations
- CASE WHEN logic
- Common Table Expressions (CTE)
- Window Functions (RANK, SUM OVER, LAG)
- Subqueries
- Date formatting and time series analysis

---

## Key Insights
- Revenue and order patterns vary significantly across product categories
- Certain payment-method and device combinations generate higher revenue
- Returning customers contribute more to total revenue
- Customer satisfaction is influenced by delivery time and category type
- Clear monthly trends and seasonal effects exist in sales performance
